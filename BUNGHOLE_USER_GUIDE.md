## Introduction

**Bunghole** is a cross-platform document aligner designed to assist professional translators in the production of translation memories from existing translated material.

Translation memories in TMX format generated by Bunghole can be used in most modern CAT (Computer Assisted Translation) tools, including [Swordfish](https://www.maxprograms.com/products/swordfish.html).

### Key Features

- **Multiple Alignment Algorithms**: Gale-Church and Hunalign for accurate sentence alignment
- **AI-Powered Improvement**: Claude AI integration for intelligent alignment correction
- **Extensive File Format Support**: 30+ file formats including Office, InDesign, XML, and more
- **Flexible Export Options**: TMX, Excel, and TAB-delimited formats
- **Real-time Editing**: Interactive alignment correction with keyboard shortcuts
- **Cost Estimation**: Transparent AI cost calculation before processing
- **Hotkey System**: Efficient navigation and editing with keyboard shortcuts
- **Cross-Platform**: Works on Windows, macOS, and Linux

## Getting Started

Alignment is a process with 3 basic steps:

1. **Create** an alignment file from source and target documents
2. **Correct** the alignment using manual editing or AI enhancement
3. **Export** the aligned segments to TMX, Excel, or TAB-delimited format

### Step 1: Create Alignment File

#### Procedure

1. Select **File ‚Üí New Alignment** or press:
   - `Ctrl+N` on Windows/Linux
   - `Cmd+N` on macOS
   - Or click the **üìÑ** button in the toolbar

2. The **New Alignment** dialog appears with the following fields:

   **Alignment File**
   - Enter name and location for the `.algn` file
   - Use **Browse...** button to select location

   **Source File** (Left Panel)
   - Select the original document
   - Choose language from dropdown
   - Select file type (auto-detected when using Browse)
   - Select character set (auto-detected when using Browse)

   **Target File** (Right Panel)
   - Select the translated document
   - Choose language from dropdown
   - Select file type (auto-detected when using Browse)
   - Select character set (auto-detected when using Browse)

3. **Optional Settings**:
   - ‚òë **Paragraph Segmentation** - Align by paragraphs instead of sentences

4. Click **Align Files** to create the alignment file

#### Results

- Segments are extracted from both documents
- Automatic alignment is performed using Gale-Church algorithm
- Hunalign is used if available for enhanced accuracy
- The alignment is displayed in the main window

### Step 3: AI-Powered Alignment Enhancement

Bunghole integrates multiple AI providers to intelligently improve alignment quality. Choose between Claude (Anthropic) or OpenAI (ChatGPT) based on your preferences.

#### Choosing an AI Provider

Bunghole supports two AI providers:

- **Claude (Anthropic)** - Default provider, excellent for nuanced translations
  - Models: Claude Sonnet 4, Opus, Haiku
  - Cost: ~$3/M input tokens, ~$15/M output tokens
  - Best for: High-quality translation analysis

- **OpenAI (ChatGPT)** - Alternative provider, widely available
  - Models: GPT-4 Turbo, GPT-4, GPT-3.5 Turbo
  - Cost: ~$10/M input tokens (GPT-4), ~$0.50/M (GPT-3.5)
  - Best for: Fast processing, budget options

See [AI_PROVIDERS.md](AI_PROVIDERS.md) for detailed comparison.

#### Prerequisites

1. **Get an API Key**:
   - **Claude**: Visit [Anthropic Console](https://console.anthropic.com/) ‚Üí Create API Key
   - **OpenAI**: Visit [OpenAI Platform](https://platform.openai.com/) ‚Üí API Keys ‚Üí Create

2. **Configure Your Provider**:
   - Go to **Settings ‚Üí Preferences** (or `Cmd+,` on macOS)
   - Select your AI provider from dropdown (Claude or OpenAI)
   - Enter your API key in the **API Key** field
   - Optionally select specific model
   - Save preferences

#### Running AI Enhancement

1. **Estimate Cost First**
   - Press `Cmd+R` / `Ctrl+R` or select **AI ‚Üí Estimate Cost**
   - View estimated token usage and cost
   - Understand what will be processed

2. **Review Cost Dialog**
   - Shows total segment pairs to analyze
   - Displays uncertain segments (low/medium confidence)
   - Shows selected AI provider and model
   - Calculates estimated cost in USD
   - Token breakdown (input/output)
   - Costs vary by provider and model selected

3. **Proceed with AI Enhancement**
   - Click **Proceed** if cost is acceptable
   - AI analyzes alignment quality
   - Confidence scores are updated
   - Uncertain segments are flagged

#### Understanding Confidence Scores

Segments are color-coded based on confidence:

- **üü¢ High Confidence** - Good alignment, likely correct
- **üü° Medium Confidence** - May need review
- **üî¥ Low Confidence** - Likely misaligned, needs attention
- **üîµ Manual Mark** - Flagged by you for review

#### Navigate Uncertain Segments

- **Next Uncertain**: Press `N` - Jump to next uncertain segment
- **Previous Uncertain**: Press `P` - Jump to previous uncertain segment
- **Mark for Review**: Hover over segment, press `M`

## Configuration

Bunghole supports externalized configuration for advanced users.

### Configuration File

Create a `config.properties` file in the application directory.

**Example Configuration:**

```properties
# Server Configuration
server.port=8040
server.bindAddress=127.0.0.1
server.minThreads=3
server.maxThreads=10
server.threadTimeout=20
server.queueSize=100

# Claude API Configuration
claude.apiUrl=https://api.anthropic.com/v1/messages
claude.model=claude-sonnet-4-20250514
claude.inputPrice=3.0
claude.outputPrice=15.0

# Application Mode
bunghole.mode=production

# Logging
logging.level=INFO
```

### Configuration Priority

Settings are loaded in this order (highest priority first):

1. **System Properties** - `-Dserver.port=8080`
2. **Configuration File** - `config.properties`
3. **Environment Variables** - `SERVER_PORT=8080`
4. **Default Values** - Built-in defaults

### Environment Variables

Override any configuration using environment variables:

```bash
export SERVER_PORT=8040
export CLAUDE_MODEL=claude-sonnet-4-20250514
export BUNGHOLE_MODE=development
```

**Conversion Rule**: Replace dots with underscores and uppercase:
- `server.port` ‚Üí `SERVER_PORT`
- `claude.apiUrl` ‚Üí `CLAUDE_APIURL`

## Security & Privacy

Bunghole takes security and privacy seriously.

### Network Security

**Localhost Only**: The internal HTTP server binds to `127.0.0.1` only, preventing network exposure.

**No Remote Access**: Your alignment data is never accessible from other computers.

**Configurable**: Can be changed via `server.bindAddress` if needed (not recommended).

### API Key Security

**Storage**: API keys (Claude and OpenAI) are stored in your application preferences.

**Transmission**: Keys are only sent to their respective providers over HTTPS:
- Claude keys ‚Üí Anthropic API (https://api.anthropic.com)
- OpenAI keys ‚Üí OpenAI API (https://api.openai.com)

**Best Practices**:
- Never commit API keys to version control
- Don't share your API keys
- Rotate keys periodically (recommended: every 90 days)
- Use separate keys for development and production
- Set spending limits in provider dashboards
- Monitor usage for unusual activity

‚ö†Ô∏è **Current Limitation**: API keys are stored in plain text in configuration. For production use, consider implementing OS keychain integration or using environment variables.

### Input Validation

Bunghole validates all file paths to prevent:
- Path traversal attacks (`../../../etc/passwd`)
- Invalid file types
- Symbolic link attacks

### Data Privacy

**Local Processing**: All alignment work is performed locally on your machine.

**AI Processing**: When using AI features:
- Segment text is sent to your selected AI provider (Claude or OpenAI)
- Provider privacy policies apply:
  - [Anthropic Privacy Policy](https://www.anthropic.com/privacy)
  - [OpenAI Privacy Policy](https://openai.com/privacy)
- No data is stored by Bunghole after processing
- No telemetry or usage tracking by Bunghole
- Review provider terms regarding data retention

**No Cloud Storage**: Alignment files remain on your local system.

**Provider Data Usage**: Be aware that:
- AI providers may use data to improve their models (check provider settings)
- Both providers offer enterprise plans with stricter data guarantees
- Consider data sensitivity when choosing to use AI features

### Security Best Practices

‚úÖ **Do**:
- Keep Bunghole updated
- Use strong API key practices
- Review alignment files before sharing
- Use localhost binding (default)
- Enable TypeScript strict mode for development

‚ùå **Don't**:
- Expose the HTTP server to network
- Share API keys or alignment files with credentials
- Process sensitive documents without reviewing privacy policies
- Disable security features

## Troubleshooting

### Common Issues

#### Alignment Quality is Poor

**Symptoms**: Many misaligned segments, incorrect pairings

**Solutions**:
1. Check that source and target files match
2. Verify language settings are correct
3. Try paragraph segmentation for documents with different sentence structures
4. Use AI enhancement to improve quality
5. Manual correction may be necessary for complex documents

---

#### AI Enhancement Fails

**Symptoms**: Error message when running AI enhancement

**Solutions**:
1. Verify API key is entered in Preferences
2. Check internet connection
3. Confirm Anthropic API is accessible
4. Review cost estimate - ensure sufficient API credits
5. Try processing smaller batches

---

#### Application Won't Start

**Symptoms**: Bunghole doesn't launch or crashes on startup

**Solutions**:
1. Check Java version (requires JDK 21+)
2. Review error logs in application directory
3. Try deleting preferences file (settings will reset)
4. Reinstall application
5. Check for conflicting software

---

#### Files Won't Open

**Symptoms**: Error when opening alignment or source files

**Solutions**:
1. Verify file format is supported
2. Check file isn't corrupted
3. Ensure file permissions allow reading
4. Try converting to a different format
5. Check file path doesn't contain special characters

---

#### Export Fails

**Symptoms**: Error when exporting TMX, Excel, or CSV

**Solutions**:
1. Check destination folder is writable
2. Close target file if already open in Excel
3. Ensure sufficient disk space
4. Verify alignment file isn't corrupted
5. Try exporting to a different location

---

### Performance Issues

#### Slow Loading

**Solutions**:
- Reduce rows per page setting
- Close other applications
- Increase available memory
- Consider splitting very large documents

#### High Memory Usage

**Solutions**:
- Process documents in smaller segments
- Close unnecessary alignment files
- Restart application periodically
- Upgrade system RAM if needed

## Glossary

### Alignment
The process of matching segments (sentences or paragraphs) in a source document with their translations in a target document.

### CAT (Computer Assisted Translation)
Computer technology applications designed to assist human translators in the translation process.

### Character Set
A character set (code page) is a collection of characters associated with numbers to facilitate text storage and transmission.

### Confidence Score
A numerical value (0-100%) indicating how certain the alignment algorithm is that two segments are correctly paired.

### CSV (Comma Separated Values)
A standard file format used to store tabular data, compatible with Excel and other spreadsheet applications.

### Gale-Church Algorithm
A statistical algorithm for aligning translated texts based on sentence length correlation.

### Hotkey
A keyboard shortcut that triggers an action without using the mouse or menu.

### Hunalign
An advanced sentence alignment tool that uses bilingual dictionaries to improve alignment accuracy.

### Segment
A portion of text, typically a sentence or paragraph, that forms a single translation unit.

### SRX (Segmentation Rules eXchange)
An XML-based open standard for describing how translation tools segment text for processing.

### TMX (Translation Memory eXchange)
An open standard XML format for exchanging translation memory data between CAT tools.

### Translation Memory (TM)
A database of source text segments and their translations, used to assist in translating similar content.

### Translation Unit (TU)
A single aligned pair of source and target segments in a translation memory.
